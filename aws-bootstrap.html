About This Guide

Create Your AWS Account

The foundation of your cloud journey. This is where you'll provide payment information and create your root account. Don't worry, AWS won't charge you for just existing (yet).

Step-by-Step Instructions:

  1. Go to aws.amazon.com and click "Create an AWS Account"
  2. Enter your email address and choose an account name (this will be your account alias)
  3. Choose "Personal" account type (you can change this later)
  4. Fill in your contact information
  5. Add a payment method (credit card required)
  6. Complete identity verification (phone call or SMS)
  7. Choose a support plan (start with "Free" tier)
  8. Wait for account activation (usually immediate)

Important Notes

‚Ä¢ Use a business email address, not a personal one
‚Ä¢ Keep your root account credentials secure - you'll rarely use them
‚Ä¢ Enable MFA on your root account immediately after creation

Set Up AWS Organizations

AWS Organizations helps you centrally manage multiple AWS accounts. Think of it as a family tree for your cloud accounts - one parent account managing multiple child accounts for different purposes.

Step-by-Step Instructions:

  1. Sign in to your AWS Management Console
  2. Search for "Organizations" in the service search bar
  3. Click "Create organization"
  4. Choose "All features" (includes consolidated billing and advanced account management)
  5. Review the terms and click "Create organization"
  6. Note your Organization ID (you'll need this later)

Pro Tip

Start with "All features" enabled. You can always disable features later, but it's harder to enable them once you have accounts in your organization.

Terraform Sample: AWS Organizations

Want to automate your AWS Organizations setup? Here's a basic Terraform example to get you started. (You'll need to run this from your management account.)

provider "aws" {
  region = "us-east-1"
}

resource "aws_organizations_organization" "main" {
  feature_set = "ALL"
}

resource "aws_organizations_account" "dev" {
  name      = "MyCompany-Dev"
  email     = "dev+aws@mycompany.com"
  parent_id = aws_organizations_organization.main.roots[0].id
}

resource "aws_organizations_account" "prod" {
  name      = "MyCompany-Prod"
  email     = "prod+aws@mycompany.com"
  parent_id = aws_organizations_organization.main.roots[0].id
}
Terraform: aws_organizations_organization | aws_organizations_account

Create Member Accounts

Now let's create some member accounts for different purposes. This is where you start thinking about account separation - development, staging, production, etc. Because putting everything in one account is like putting all your eggs in one basket, except the basket is on fire.

Step-by-Step Instructions:

  1. In AWS Organizations console, click "Accounts" ‚Üí "Create account"
  2. Create accounts for:
    • Development - for development work
    • Staging - for testing before production
    • Production - for live applications
    • Security - for security tools and monitoring
  3. For each account, provide:
    • Account name (e.g., "MyCompany-Dev")
    • Email address (must be unique and valid)
    • IAM role name (default is fine)
  4. Wait for account creation (5-10 minutes per account)
  5. Note the account IDs and email addresses

Account Naming Convention

Use a consistent naming convention like: [Company]-[Environment]-[Purpose]
Example: MyCompany-Dev-Apps, MyCompany-Prod-Data

Configure IAM Identity Center

IAM Identity Center (formerly AWS SSO) is your central identity provider. It's like having one key that opens all the doors in your cloud mansion, but with proper security controls.

Step-by-Step Instructions:

  1. In your management account, search for "IAM Identity Center"
  2. Click "Enable" to set up IAM Identity Center
  3. Choose your identity source:
    • Identity Center directory (recommended for startups)
    • Active Directory (if you have on-prem AD)
    • External identity provider (if using Okta, Azure AD, etc.)
  4. Create your first user (yourself)
  5. Create a permission set for administrators
  6. Assign the permission set to your user and accounts
  7. Set up MFA for your user

Permission Set Strategy

Create permission sets like: Administrator, Developer, ReadOnly. Start with Administrator for yourself, then add others as needed.

Terraform Sample: IAM Identity Center (SSO)

Here's a basic example to create an IAM Identity Center user and permission set. (Requires AWS provider v5+.)

provider "aws" {
  region = "us-east-1"
}

resource "aws_identitystore_user" "admin" {
  identity_store_id = data.aws_ssoadmin_instances.main.identity_store_ids[0]
  user_name         = "jason"
  display_name      = "Jason Merefield"
  emails {
    value = "jason@mycompany.com"
    primary = true
  }
}

resource "aws_ssoadmin_permission_set" "admin" {
  name             = "AdministratorAccess"
  instance_arn     = data.aws_ssoadmin_instances.main.arns[0]
  description      = "Admin access for all accounts"
  session_duration = "PT8H"
  managed_policies = ["arn:aws:iam::aws:policy/AdministratorAccess"]
}
Terraform: aws_identitystore_user | aws_ssoadmin_permission_set

Set Up AWS Control Tower (Optional)

Control Tower is like having a security guard, compliance officer, and auditor all rolled into one automated service. It's more complex to set up but provides enterprise-grade governance from day one.

Step-by-Step Instructions:

  1. Ensure you have a clean Organizations setup (no existing member accounts)
  2. Search for "Control Tower" in the AWS console
  3. Click "Set up Control Tower"
  4. Choose your home region (where Control Tower will be deployed)
  5. Configure your organizational units (OUs):
    • Security OU (for security accounts)
    • Sandbox OU (for development/testing)
  6. Set up your audit and logging accounts
  7. Review and launch Control Tower
  8. Wait for deployment (30-45 minutes)

Important Considerations

‚Ä¢ Control Tower requires a clean Organizations setup
‚Ä¢ It will create additional accounts for logging and audit
‚Ä¢ Some regions may not support all Control Tower features
‚Ä¢ Consider this for companies with compliance requirements
